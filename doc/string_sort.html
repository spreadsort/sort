Introduction

string_sort is a hybrid radix-based/comparison-based algorithm that sorts strings of characters in ascending order. The simplest version (no functors) sorts strings of items that can cast to an unsigned data type (such as an unsigned char), have a < operator, and have a size function, such as a std::string. The functor version can sort any data type that has a strict weak ordering, via templating, but requires definitions of a get_char (acts like x[offset] on a string or a byte array), get_length (returns length of the string being sorted), and a comparison functor. Individual characters returned by get_char must support the != operator and have an unsigned value that defines their lexicographical order. This algorithm is not efficient for character types larger than 2 bytes each, and is optimized for one-byte character strings. For this reason, std::sort will be called instead if the character type is of size > 2. string_sort has a special optimization for identical substrings. This adds some overhead on random data, but identical substrings are common in real strings. reverse_string_sort sorts strings in reverse (decending) order, but is otherwise identical. string_sort is sufficiently flexible that it should sort any data type that std::sort can, assuming the user provides appropriate functors that index into a key.

Requires: For the first version of string_sort, the one that takes three arguments:
  • [first, last) is a valid range.
  • RandomAccessIterator is a Random Access Iterator.
  • RandomAccessIterator is mutable.
  • RandomAccessIterator's value type is LessThan Comparable.
  • RandomAccessIterator's value type supports .size(), which returns the integer length of the value type.
  • RandomAccessIterator's value type supports the [] operator, which returns an integer data type that is castable to and the same size as Unsigned_char_type. Given that the leading n - 1 characters are the same for a and b, (Unsigned_char_type)a[n] < (Unsigned_char_type)b[n] -> a < b.

Postconditions: The elements in the range [first, last) are sorted in ascending order.
Complexity: O(N * mean_string_length) character comparisons worst-case performance, where N is last - first.
Exceptions: Throws if any of the element comparisons, the element swaps (or moves), one of the functors, subtraction of the characters obtained through get_char, or the operations on iterators throws. Throwing an exception may cause data loss. Note that invalid arguments cause undefined behavior.
Rationale:

This provides a general-case string-sorting algorithm that takes asymptotically less time than is possible for a pure comparison-based algorithm.

Why use recursion for string_sort instead of a stack, like American Flag Sort?

Because recursion is simpler, more readable, and has comparable performance. The stacksize limitation is not a serious issue on common modern systems.

Case-insensitive sorting 

vector<string> vec;
vec.push_back("c");
vec.push_back("ab");
vec.push_back("a");
string_sort(vec.begin(), vec.end());
//The sorted vector contains "a ab c"

Reverse order:

struct greaterthan { 
    bool operator()(const DATATYPE &x, const DATATYPE &y) const 
    { return x > y; } 
};
reverse_string_sort(vec.begin(), vec.end(), std::greater<string>());
//The sorted vector contains "c ab a"

Structure:

struct data_type {
	string a;
	inline bool operator<(const data_type &y) const { return a < y.a;}
};
vector<data_type> vec;

struct bracket {
	unsigned char operator()(const data_type &x, size_t offset) const { return x.a[offset]; }
};
struct getsize {
	size_t operator()(const data_type &x) const{ return x.a.size(); }
};

string_sort(vec.begin(), vec.end(), bracket(), getsize());
