Introduction

spreadsort checks whether the data type provided is an integer, castable float, string, or wstring. If it's an integer, integer_sort is used. If it's a float, float_sort is used. If it's a string or wstring, string_sort is used. Sorting other data types requires picking between integer_sort, float_sort and string_sort directly, as spreadsort won't accept types that don't have the appropriate type traits.

Functions

spreadsort
template 
  inline void spreadsort(RandomAccessIter first, RandomAccessIter last);
Requires:
  • [first, last) is a valid range.
  • RandomAccessIterator is a Random Access Iterator.
  • RandomAccessIterator is mutable.
  • RandomAccessIterator's value type is LessThan Comparable.
  • RandomAccessIterator's value type supports the >> operator, which returns an integer data type rightshifted a specified number of bits.
Postconditions: The elements in the range [first, last) are sorted in ascending order.
Complexity: The lesser of comparisons and operations worst-case, where N is last - first, K is the log of the range in bits (32 for 32-bit integers using their full range), and S is a constant called max_splits, defaulting to 11, except for strings, where it is the log of the character size.
Exceptions: Throws if any of the element comparisons, the element swaps (or moves), the right shift, subtraction of right-shifted elements, functors, or the operations on iterators throws. Throwing an exception may cause data loss. This will also throw if a small vector resize throws, in which case there will be no data loss. Note that invalid arguments cause undefined behavior.
Rationale: This provides a wrapper that calls the fastest sorting algorithm available for a data type, enabling faster generic programming.

Simple example sorting floats 

vector<float> vec;
vec.push_back(1.0);
vec.push_back(2.3);
vec.push_back(1.3);
spreadsort(vec.begin(), vec.end());
//The sorted vector contains "1.0 1.3 2.3"
